实验一 在Github上用Markdown创建技术博客

一、实验目的

  • 1.熟悉Github网站
  • 2.掌握Markdown语法

二、实验任务

  • 1.注册Github账号
  • 2.创建Gihub Pages repository
  • 3.发布第一篇博客
  • 4.学习Markdown
  • 5.利用Hexo生成博客并部署到Github

三、实验步骤及结果

  • 1.访问github网站，创建个人账户。
  • 2.开始创建我的第一个博客
  • 3.发布第一篇博客，和同学互相访问各自的账户
  • 4.学习Markdown：
  • 5.访问zybuluo.com,学习现成的语法规则。
  • 6.安装 node-v7.7.1-x86.msi
  • 7.安装 Git-2.1.2.0-32-bit.exe
  • 8.电脑 win+R 输入 cmd 回车，执 行 npm install -g cnpm –registry=https://registry.npm.taobao.org命令
  • 9.在命令窗口中直接执行 cnpm install -g hexo-cli
  • 10.在cmd命令窗口中，创建MD文件夹，执行语句为：mkdir MD
  • 11.打开MD，创建hexo文件夹：执行命令为：cd /d D:\MD,回车，执行：mkdir hexo
  • 12.打开hexo文件夹，执行hexo init hexo,执行语句为：cd /d D:\MD\hexo,回车，执行hexo init hexo 语句
  • 13.执行 cnpm install
  • 14.执行 hexo server

四、实验小结

  • 通过对Github网站的学习和了解还有对Markdown语法的掌握，我知道创建博客是为了在学习时，通过博客记录，更多的了解自己所学到的内容，并进行巩固。在创建Gihub Pages repository时，要注意名称必须是用户名.github.io。还有利用Hexo生成博客并部署到Github时可以先在本地查看Hexo.以上就是我对这次实验的总结。
